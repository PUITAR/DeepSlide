Introduction: Vector Embeddings and Similarity Search

  • Vector embeddings convert words and sentences into numerical vectors
  • Capture semantic meaning and relationships between concepts
  • Example: $\heartsuit \overset{\text{embedding}}{\implies} [0.124, 1.564, ...]$
  • AI systems need to find vectors most similar to query vectors
  • This is called Nearest Neighbor Search (NNS)

Key Insight: Smaller distance = Higher similarity between vectors







  

    

      
Introduction to ANNS

      

    

    

  


  

    

      

        
    
            
  • Exact NNS is computationally expensive for large datasets
    • 
            
  • Approximate Nearest Neighbor Search (ANNS) finds close matches efficiently
    • 
            
  • Mildly relaxes accuracy constraints for speed
    • 
            
  • Essential for real-world AI applications:
            
      
                
    • Information retrieval zhu2019accelerating
      • 
                
    • Pattern recognition kosuge2019object
      • 
                
    • Retrieval Augmented Generation (RAG) brown2020language
      • 
            
    • 
            
  • Four main ANNS approaches: tree-based, hashing-based, quantization-based, and graph-based
    • 
        


        
Graph-based approaches offer high quality and low latency

Graph-Based ANNS: Core Concepts

      

    

    

  


  

    

      

        
    
        
  • Graph Definition: $G = (V, E)$ where $V$ are nodes (vectors) and $E$ are edges (connections)
    • 
        
  • Six Degrees of Separation: Any two nodes can be connected through at most 6 intermediate nodes
    • 
        
  • Proximity Graphs: Connect similar vectors to enable efficient search
    • 
        

        
    
        
  • Key Insight: Graph structure enables finding nearest neighbors by traversing connections
    • 
        
  • Challenge: Balancing search speed vs. accuracy in high-dimensional spaces
    •

Current Limitations & Pain Points

      

    

    

  


  

    

      

        
    
                  
  • Traditional Methods: Struggle with computational efficiency during nearest neighbor searches
    • 
                  
  • Speed-Accuracy Trade-off: Existing approaches can't optimize both simultaneously
    • 
                  
  • Graph Construction: Most methods use both long and short edges:
                    
      
                      
    • Long edges: For quick initial neighbor discovery (early stage)
      • 
                      
    • Short edges: For precise late-stage refinement
      • 
                    
    
                  
    • 
                  
  • Fixed Parameters: Current algorithms use uniform search strategies throughout the entire process
    • 
                  
  • Key Insight: Search process should be divided into stages with different optimization strategies
    • 
                

                
State-of-the-art: HNSW malkov2018efficient and other proximity graphs dobson2023scaling show promise but need improvement in dynamic optimization.

CSPG: Cross-Partition Search on Proximity Graphs

      

    

    

  


  

    

      

        
    
                  
  • Core Idea: Partition dataset into smaller graphs to reduce distance computations
    • 
                  
  • Key Innovation: Two-stage search balancing speed and accuracy
    • 
                  
  • Goal: Faster search while maintaining high accuracy
    • 
                  
  • Approach: Random partitioning + shared routing vectors
    • 
                

                
"When far from query: use large steps for speed
When close to query: use small steps for accuracy"

      

    


    

      


      

    

  


  






  

    

      
Partitioning Algorithm

      

    

    

  


  

    

      

        
    
            
  • Randomly partition dataset \(\mathcal{D}\) into multiple groups
    • 
            
  • Build sparse, smaller graphs for each group (\(\mathcal{G}_1\), \(\mathcal{G}_2\))
    • 
            
  • Shared routing vectors (red) connect different partitions
    • 
            
  • Group-specific vectors: green for \(\mathcal{G}_1\), blue for \(\mathcal{G}_2\)
    •

Two-Stage Search Algorithm

      

    

    

  


  

    

      

        
    
            
  • Stage 1: Fast Approaching
            
      
                
    • Greedy beam search using only one partition
      • 
                
    • Small candidate set size $ef_1$ for speed
      • 
                
    • Quickly zones into query's nearby region
      • 
            
    • 

            
  • Stage 2: Precise Search
            
      
                
    • Cross-partition expansion with larger candidate set $ef_2$
      • 
                
    • When encountering routing vectors, add all instances from all partitions
      • 
                
    • Maximizes search space for final closest vectors
      • 
            
    • 

            
  • Key Parameters: $ef_1 < ef_2$ for optimal speed-accuracy balance
    • 
        

      

    


    

      

        
      

    

  


  






  

    

      
CSPG Method Summary

      

    

    

  


  

    

      

        
    
                  
  • Partitioning: Random dataset division with shared routing vectors
    • 
                  
  • Search: Two-stage approach (fast approaching + precise search)
    • 
                  
  • Advantage: Reduces distance computations while maintaining accuracy
    • 
                  
  • Innovation: Cross-partition expansion via routing vectors
    • 
                

                
Next: Experimental evaluation showing performance improvements

      

    


    

      


      

    

  


  






  

    

      
CSPG: Key Contributions

      

    

    

  


  

    

      

        
    
                    
  • Novel ANNS Schema: Crossing Sparse Proximity Graph (CSPG) framework
    • 
                    
  • Random Partitioning: Dataset split into groups with separate, sparser graphs
    • 
                    
  • Routing Vectors: Shared vectors enable cross-partition traversal
    • 
                    
  • Two-Stage Search:
                    
      
                        
    • Fast greedy beam search for initial approach
      • 
                        
    • Cross-partition expansion for precise results
      • 
                    
    
                    
    • 
                    
  • Key Advantage: Achieves both higher speed and higher accuracy
    • 
                

                
Core Innovation: Random partitioning creates smaller, sparser graphs that enable faster traversal while maintaining accuracy through intelligent cross-partition search.

      

    


    

      


      

    

  


  






  

    

      
Experimental Results: Higher QPS & Recall@10

      

    

    

  


  

    

      

        
    
                  
  • Consistent Improvement: Higher QPS and Recall@10 across all datasets
    • 
                  
  • Significant Speedup: 1.5x+ speedup for Vamana and HCNNG indices
    • 
                  
  • Better Pareto Frontiers: Optimal trade-off between speed and accuracy
    • 
                  
  • Validated Framework: CSPG enhances existing graph-based ANNS algorithms
    • 
                

                
Conclusion: CSPG achieves both improved efficiency and accuracy in nearest neighbor search.
