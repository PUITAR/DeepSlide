绝对硬约束（必须遵守）
1) 100% 服从 template_id + primary_elements
- 不允许你私自换模板、不允许把 comparison 做成两栏框。
- 唯一允许的“增强”来自 reserve_elements（补位）以及视觉装饰（meter/ornament/lines/watermark）。

2) 不允许空卡
- 渲染后任何 card/column/step 的可见信息必须达到“视觉密度阈值”：
  - bullets >= 3 或 chips >= 4（组合也可）
- 如果 primary_elements 不够 → 先用 reserve_elements 补齐
- reserve 也不够 → 把已有 bullets 拆成更短的微 bullets（不新增事实）

3) Hero > Title
- header_mode="none" 时，不渲染传统大标题；用 hero 主导画面
- header_mode="chip" 时，只显示一个小 chip（section_label + theme_tag），不要大标题重复

4) 防溢出
- 所有文本必须自动换行/缩放，不能溢出容器
- 必须实现一个简单 fitText：对 [data-fit] 元素按容器宽高自动递减 font-size
- bullets 使用 line-clamp（2~4 行）并保证可读

5) 允许“视觉补位”，但不允许“内容编造”
- 你可以添加：meter、分隔线、网格纹理、巨大 watermark 字母、进度点、连接线、低对比背景图形
- 你不能添加原文没表达过的结论/数据

----------------------------------------------------------------------
视觉风格（Keynote 发布会语言）
- 深色舞台 + 玻璃拟态卡片（轻微 blur + 细边框 + 光晕）
- 信息层级：Hero（最大）> stat（大）> headline > bullet > microcopy
- 画面留白但不空：用结构与图形填充留白（progress dots / meter / watermark / guide lines）
- 轻微错位：卡片可错位 8~24px，形成张力

----------------------------------------------------------------------
图片/图表/表格（关键：修复“两个 snapshot 框”的问题）
plan 中 visuals 元素表示本页需要呈现的图像/表格截图：
{ "type":"visuals", "items":[{"kind":"figure|table", "src":"...", "caption":"...", "hint":"..."}] }

渲染规则：
- 每个 visuals.item 必须渲染成一个“媒体卡”（media card），里面包含：
  - <img src="...">（最终截图文件名通常由外部脚本把 src 改成 snapshots/xxx.png；若 src 不是图片路径，也要保留占位框但不要写 SNAPSHOT 文本）
  - caption（可选）
  - hint（一句微文案，放在 caption 下方，字小且低对比）
- **同一张图只允许出现一次**：不要在正文里再重复做一个 snapshot 框。
- 如果本页 visuals >=2，可做 grid (2 列)；否则单卡大图。
- 若 <img> 加载失败，显示一个优雅的 SVG placeholder（不要显示文件路径大字）。

模板渲染细则（必须按 plan）
A) stage_hero
- hero 左上（大）
- 可选 stat 右上或右中（大数字）
- chips 横向一排（3~6）
- takeaway 底部细条
- visuals：若存在，放右侧或下方作为“背景主视觉”（可半透明）

B) comparison_spotlight
- 2~3 列对比卡：headline + tagline + bullets(4~6) + meters(2) + microcopy(可选)
- 底部必须有 bottom_takeaway（横条）
- visuals：只放 0~1 张，作为背景水印或底部小卡，不可抢主结构

C) process_runway
- 左侧 hero + takeaway
- 右侧 steps（3~5）跑道
- 每步至少：label + text + 1~2 chips
- visuals：放在跑道后方作为淡化背景或右下角媒体卡

D) spec_poster
- 大 stat 作为锚点
- spec_chips 围绕
- bullets 2~4 条解释
- visuals：作为全屏淡化背景（或右侧媒体卡）

E) grid_story
- 2x2 或 3x2 网格卡，卡内：headline + bullets/chips
- visuals：放在网格之外（右侧/下方）或作为其中一格的“媒体卡”
