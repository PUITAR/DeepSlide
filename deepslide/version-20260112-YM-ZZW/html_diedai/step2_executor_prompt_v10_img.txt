【新增：Snapshot 渲染规则（必须遵守）】
A) Snapshot 来源与识别
- snapshot 在 step1_generated.html 里以 figure-box 出现：
  <div class="figure-box" data-snapshot-id="SNAPSHOT_1"> ... <img class="snapshot-img" src="data:image/png;base64,...">
- 你的输出必须保留这些图片（直接复制 data URI 的 src），不要引用外链，不要重新截图。

B) Snapshot 与 plan 的对齐
- 如果 plan 的 primary_elements 中存在：
  { "type":"diagram_spec", "mode":"snapshots", "items":[...], "layout_hint":"..." }
  你必须把 items 中每个 snapshot_id 对应的图片从 step1 找出来并渲染到该页：
  - layout_hint="right": 放到右侧媒体卡/图像位
  - layout_hint="bottom": 放到底部横向媒体条
  - layout_hint="center": 作为画面中心的大图（可带玻璃框）
- 如果 plan 没写 diagram_spec，但 step1 里该页确实有 snapshot：你仍然必须保留（自动放入该页的媒体区域，作为“补位信息块”）。

C) 画面密度与版式
- Snapshot 本身可以算“信息块”，但仍需满足“不空卡”规则：文字不足时用 reserve_elements 补齐。
- Snapshot 容器必须有：标题/小 badge（可用 snapshot_id 或 “Figure”）、可选 caption（来自 plan 或留空）。

绝对硬约束（必须遵守）
1) 100% 服从 template_id + primary_elements
- 不允许你私自换模板、不允许把 comparison 做成两栏框。
- 你唯一可以做的“增强”来自 reserve_elements（补位）以及视觉装饰（meter/ornament/lines）。

2) 不允许空卡
- 渲染后任何 card/column/step 的可见信息必须达到“视觉密度阈值”：
  - bullets >= 3 或 chips >= 4（组合也可）
- 如果 primary_elements 不够 → 先用 reserve_elements 补齐
- 如果 reserve 也不够 → 把已有 bullets 拆成更短的微 bullets（不新增事实）

3) Hero > Title
- header_mode="none" 时，不渲染传统标题；用 hero 主导画面
- header_mode="chip" 时，只显示一个小 chip（section_label + theme_tag），不要大标题重复

4) 防溢出
- 所有文本必须自动换行/缩放，不能溢出容器
- 必须实现一个简单 fitText：对 [data-fit] 元素按容器宽高自动递减 font-size
- bullets 使用 line-clamp（2~4 行）并保证可读

5) 允许“视觉补位”，但不允许“内容编造”
- 你可以添加：meter、分隔线、网格纹理、巨大 watermark 字母、进度点、连接线、低对比背景图形
- 你不能添加原文没表达过的结论/数据

----------------------------------------------------------------------
渲染规范（发布会语言）
- 画面结构：大留白 + 少量高对比信息块（但必须“看起来被设计过”）
- 信息层级：Hero（最大）> stat（大）> headline > bullet > microcopy
- 错位与张力：卡片不要完全齐平，可轻微错位（8~24px），并用光晕/边框形成层次
- 对比模板：comparison_spotlight 必须渲染 meters（2 条即可，level 1~5 显示为条形长度）

模板渲染细则（必须按 plan）
A) stage_hero
- hero 放左上（大）
- 可选 stat 放右上或右中（大数字）
- chips 横向一排（3~6 个）
- takeaway 放底部一条（细）

B) comparison_spotlight
- 2~3 列卡片，列内结构固定：
  headline（大） + tagline（小） + bullets（4~6） + meters（2 条） + （可选）microcopy
- 底部必须有 bottom_takeaway（横条）
- 如果某列 bullets < 4：先从 reserve 拿；不够就拆句补齐

C) process_runway
- 左侧 hero + takeaway
- 右侧 steps（3~5）纵向/横向 runway
- 每步必须至少：label + text + 1~2 chips

D) spec_poster
- 大 stat（极大）作为画面锚点
- spec_chips 一圈/一排围绕 stat
- bullets 2~4 条解释

E) grid_story
- 2x2 或 3x2 网格卡，卡内：headline + bullets/chips
- 网格必须“看起来满”：卡片数量>=4；每卡内容不空
