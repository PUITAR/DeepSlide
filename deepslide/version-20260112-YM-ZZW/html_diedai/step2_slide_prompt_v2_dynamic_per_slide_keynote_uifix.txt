IMPORTANT VISUAL FIXES YOU MUST APPLY
A) Bullets marker: ONLY ONE symbol
- Render bullets as:
  <ul class="bullets"> <li>Pure text only...</li> ... </ul>
- DO NOT put leading symbols inside the text (no "•", no "→", no "-", no numbering).
- The arrow marker is handled by CSS ::before in the shell.

B) Callout pills (fix: box big, text small)
- If a slide needs 2 short opposite concepts (e.g., "Smaller distance" vs "Higher similarity"),
  render them as callout pills that hug text:
  <div class="callout-row">
    <span class="callout-pill" data-fit>Smaller distance</span>
    <span class="callout-line" aria-hidden="true"></span>
    <span class="callout-pill" data-fit>Higher similarity</span>
  </div>
- DO NOT render them as large empty cards.

C) Divider (fix: double white lines)
- If you need a separator, use ONLY:
  <div class="divider" aria-hidden="true"></div>
- Do NOT create multiple <hr> or stacked lines.

D) Math (fix: formula looks broken)
- For inline math tokens in Step1 like $G=(V,E)$ or \(G=(V,E)\),
  keep the content BUT render as:
  <span class="math">G=(V,E)</span>
  (strip the $ or \( \) wrappers; do not add new macros)

E) KPI number emphasis (fix: highlight key numbers only)
- Use plan.hero.emphasis list as the ONLY source for highlight targets.
- Wherever a substring exactly matches an emphasis item, wrap it with:
  <span class="kpi">...</span>
- Do not highlight other numbers.
