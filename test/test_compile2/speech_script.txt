Today I'll introduce our CSPG method - Cross-Partition Sparse Proximity Graph. Our goal is to improve graph-based approximate nearest neighbor search by using a partitioning strategy to reduce the number of distance computations. The core insight is that we can maintain accuracy while dramatically improving speed. Our approach has two key algorithms: a partitioning algorithm and a search algorithm, which work together in a two-stage process that combines fast initial search with precise final exploration.<next>
Let's start with the partitioning algorithm. We randomly divide the dataset into multiple groups, each with its own sparse proximity graph. The key innovation is that we have shared routing vectors that connect different partitions. These routing vectors are common to all groups and serve as bridges between partitions. Because each subgraph is sparser than a full dataset graph, we can take larger steps when we're far from the query, which speeds up the initial search. The routing vectors then enable efficient fine-grained search when we get close to the target.<next>
Now let's look at the search algorithm, which uses a two-stage approach. In the first stage, we perform a greedy beam search on just one partition to quickly approach the query's nearby region. This uses a smaller expansion factor for speed. Once we're close, the second stage begins. Here, if we encounter a routing vector, we expand to include all its instances across all partitions. This allows dynamic traversal between different proximity graphs, maximizing our search space to find the closest vectors. The key is that the second stage appears to search within a single graph, but actually explores multiple partitions.<next>
To summarize, CSPG combines random partitioning with a two-stage search. The partitioning creates sparse subgraphs with shared routing vectors. The search first quickly approaches the query using one partition, then expands across all partitions when routing vectors are encountered. This dynamic traversal across multiple sparse graphs gives us both speed and accuracy advantages over traditional proximity graph methods. With appropriate partitioning, CSPG consistently outperforms existing approaches in both query speed and recall rates.<next>
In conclusion, I've proposed a novel schema for approximate nearest neighbor search called CSPG - Crossing Sparse Proximity Graph. The key innovation is using random dataset partitioning to create smaller, sparser proximity graphs for each partition, along with routing vectors that connect them. I designed an efficient two-stage search approach that combines fast proximity search with cross-partition precise expansion. This allows my method to approach queries quickly using sparser graphs, then collect results precisely by crossing different partitions when encountering routing vectors. The goal is clear: achieve higher QPS and higher Recall@10 than traditional proximity graph implementations.<next>
The experimental results demonstrate CSPG's superior performance. I tested on four standard benchmark datasets: SIFT1M, GIST1M, DEEP1M, and SIFT10M. I compared against leading graph-based ANNS algorithms including HNSW, HCNNG, Vamana, and NSG. The key metrics are QPS - queries per second for speed, and Recall@10 for accuracy. This means CSPG delivers both higher speed and higher accuracy simultaneously.<next>
In summary, CSPG works because random partitioning creates smaller, sparser graphs that are more efficient to search. The two-stage approach combines fast initial search with precise cross-partition expansion. Routing vectors enable efficient exploration across different partitions. The result is higher QPS for speed and higher Recall@10 for accuracy. This performance improvement is consistent across multiple datasets and baseline algorithms, demonstrating CSPG's effectiveness as a superior ANNS method.
