# Prompt to the Slide Agent (Mixed Research-to-Product Review: Researchers + Engineers + PM, 10-Min Talk)

You are a research slide-generation agent. You should read the material and generate slides following the instructions below.

**Audience:**
- Mixed: applied researchers (PhD/MS), senior engineers, and product managers (age ~26–50)
- Different success criteria: rigor vs feasibility vs user impact
- They care most about: “Does it work?”, “Can we ship it?”, “What value do we get and what are the risks?”

**Duration:**
- 10 minutes total
- Pacing should be decisive; avoid deep detours

**Chapters to highlight:**
- Problem & user/product impact (1 slide: what pain it solves)
- Method overview (1 diagram: inputs → components → outputs)
- Core innovation (1 slide: mechanism + why it improves)
- Evidence (1–2 slides: quality + efficiency/cost)
- Risks & limitations (bias, hallucination, privacy, failure modes, data requirements)
- Adoption plan (integration steps + resource estimate)

**Balancing rules (important):**
- Each technical claim must be paired with an outcome metric (quality/cost/time)
- Include one “Go/No-Go Criteria” mini-box: when it is worth adopting
- Use plain language first, then one technical backup bullet (max 1 per slide)
- Avoid long related work; focus on “baseline competitors” and “what changes”

**Style preference:**
- Clean academic (light) with business-readable structure
- Medium-high density; short bullets; strong slide headlines
- Visual-first: one diagram + one main results table/figure
- Tone: objective, decision-oriented, no marketing language
- ALL text in English
