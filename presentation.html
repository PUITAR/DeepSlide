Now let me introduce our novel approach called CSPG, which stands for Cross-Partition Sparse Proximity Graph. The core idea is to use random partitioning to create smaller, sparser graphs for faster search. As shown in this diagram, we randomly divide the dataset into groups that share some common vectors called routing vectors, shown in red. Each group then builds its own sparser proximity graph. The green vectors belong to group one, and blue vectors belong to group two. The key insight is that sparse graphs enable larger steps for a fast approach, while routing vectors enable precise search.

The search process has two stages. First, we perform a fast approaching stage using greedy beam search within a single partition's sparse graph. We use a small candidate set size, called ef₁, for maximum speed. This quickly gets us close to the query's neighborhood. Then, when we encounter routing vectors during this search, we trigger the second stage: cross-partition expansion. Here, we add all instances of that routing vector from other partitions to our candidate set, using a larger size ef₂ for accuracy, which enables precise search across partitions. This reduces distance computations while maintaining accuracy.
