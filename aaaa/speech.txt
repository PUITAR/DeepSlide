Let's start with the fundamentals of graph-based approximate nearest neighbor search. A graph is simply a set of nodes connected by edges. In our context, each node represents a data vector, and edges connect similar vectors. This structure is inspired by the famous Six Degrees of Separation theorem, which suggests that any two people in the world can be connected through at most six intermediate acquaintances. Similarly, in graph-based ANNS, we aim to connect vectors so that we can find nearest neighbors by traversing connections in a proximity graph. The challenge lies in balancing search speed versus accuracy in high-dimensional spaces.

Now, let's examine the current limitations. Traditional graph-based methods face a fundamental trade-off: they struggle to balance computational efficiency with search accuracy. Most existing approaches, like HNSW, construct graphs with both long and short edges. Long edges help quickly locate the general neighborhood of a query vector, while short edges enable precise refinement. However, these methods typically use fixed search parameters throughout the entire process, which limits optimization. The key insight is that the search process should be divided into stages with different strategies. State-of-the-art methods show promise but need improvement in dynamic optimization.

Today I'll introduce our novel CSPG method - that's Cross-Partition Search on Proximity Graphs. This is a graph-based approach for approximate nearest neighbor search that aims to significantly improve performance through a clever partitioning strategy. The core idea is simple but powerful: we partition the dataset into smaller groups and build separate proximity graphs for each. This reduces the number of distance computations needed during search, which is the main computational bottleneck. Our key innovation is a two-stage search balancing speed and accuracy. The goal is faster search while maintaining high accuracy through random partitioning and shared routing vectors. When far from the query, we use large steps for speed; when close, we use small steps for accuracy.
<next>
Let's look at the partitioning algorithm. As shown in this diagram, we randomly divide the dataset into multiple groups - here we see two groups as an example. For each group, we build a separate, sparser proximity graph. These are labeled as G1 on the left and G2 on the right. The key innovation is the shared routing vectors shown in red. These vectors exist in multiple partitions and serve as bridges between different groups. They enable efficient cross-partition communication. The group-specific vectors are shown in green for G1 and blue for G2.
<next>
Now let's examine the two-stage search algorithm. The first stage is fast approaching, where we conduct a greedy beam search using only one partition. We use a small candidate set size, denoted as ef1, to maximize speed. This stage quickly zones into the query's nearby region. The second stage is precise search. Here we use cross-partition expansion with a larger candidate set size ef2. The key difference is that when we encounter a routing vector during expansion, we add all its instances from all partitions. This maximizes the search space for finding the final closest vectors. The key parameters require ef1 to be less than ef2 for optimal speed-accuracy balance.

To summarize, our CSPG method introduces two key innovations: first, a partitioning algorithm that randomly divides the dataset into smaller groups with shared routing vectors, and second, a two-stage search algorithm that balances speed and accuracy through fast approaching followed by precise cross-partition expansion. The main advantage is the reduction in distance computations while maintaining high accuracy. The cross-partition expansion via routing vectors is a novel approach that allows us to maximize the search space. Next, we will look at experimental evaluation showing performance improvements.

In conclusion, I've proposed a novel schema for approximate nearest neighbor search called CSPG - the Crossing Sparse Proximity Graph. The key innovation is using random partitioning to divide the dataset into groups, each with its own smaller and sparser proximity graph, along with shared routing vectors. This enables a two-stage search approach: first, a fast greedy beam search quickly approaches the query region using one sparse graph, and then cross-partition expansion dynamically traverses multiple partitions for precise results. The core advantage is achieving both higher speed and higher accuracy by creating smaller, sparser graphs that enable faster traversal while maintaining accuracy through intelligent cross-partition search.
